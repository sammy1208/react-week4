# AO3 匯出檔轉 Nyarchive Markdown 修改手冊

本手冊依據 `docs/範本` 的檔案格式，比對 `docs/比對` 內目前的 AO3 匯出檔，整理出需要修改的規則與步驟。

## 目標格式

整理後的小說 Markdown 應該只保留兩種內容：

1. 檔案最上方的 frontmatter。
2. 小說正文與必要的章節區塊。

AO3 匯出的完整 HTML 外殼不需要保留，包括 `<!DOCTYPE>`、`<html>`、`<head>`、`<body>`、`style`、`preface`、AO3 留言連結等。

標準 frontmatter：

```md
---
title: 作品標題
author: 作者名稱
summary: |
  > 摘要文字
---
```

## 範本規格

### 單章作品

參考：`docs/範本/浴室裡的相遇.md`

單章作品不需要 `### Chapter 1`，直接保留正文容器：

```md
---
title: 【配對】作品標題
author: 作者名稱
summary: |
  > 摘要文字
---

<div id="chapters" class="userstuff">
  <div class="userstuff">
    <p>第一段正文。</p>
    <p>第二段正文。</p>
  </div>
</div>

> End Notes / 後記
> 後記內容
```

### 多章作品

參考：`docs/範本/範本01.md`

多章作品每章使用 `### Chapter N`，章節正文包在 `<!--chapter content-->` 和 `<!--/chapter content-->` 中：

```md
---
title: 【配對】作品標題
author: 作者名稱
summary: |
  > 摘要文字
---

### Chapter 1

<!--chapter content-->
<div>
  <p>第一章正文。</p>
</div>
<!--/chapter content-->

### Chapter 2

<!--chapter content-->
<div>
  <p>第二章正文。</p>
</div>
<!--/chapter content-->
```

## 通用修改規則

### 1. 移除 HTML 外殼

從比對檔刪除下列區塊：

- `<!DOCTYPE ...>`
- `<html ...>` 到 `<body>` 之前的整段 `<head>`。
- 最後的 `</body>`、`</html>`。
- `<style>` 內所有 CSS。

### 2. 將 AO3 preface 轉成 frontmatter

AO3 的 `<div id="preface">` 不保留原樣，只抽出資料：

- `<h1>` 轉成 `title`。
- `<div class="byline">by ...</div>` 轉成 `author`。
- `<blockquote class="userstuff">` 中的 Summary 轉成 `summary`。

不需要保留：

- Rating
- Archive Warning
- Category
- Fandom
- Relationship / Relationships
- Characters
- Additional Tags
- Language
- Stats
- Posted originally on...
- AO3 原文連結

### 3. 清理正文中的 AO3 雜訊

AO3 匯出常出現這種段落：

```html
<p>
  <span>正文。</span>
  <span> </span>
</p>
```

整理成：

```html
<p>正文。</p>
```

需要移除：

- 空白 `<span> </span>`
- 只有 `?` 或空白符號的 `<span>`
- 多餘縮排
- 多餘空白行
- `<h2 class="toc-heading">...</h2>`

### 4. 保留正文 HTML

正文裡可以保留這些標籤：

- `<p>`
- `<br>`
- `<strong>`
- `<em>`
- `<mark>`
- `<div>`

如果 AO3 原文裡有 `<span>`，原則上要拿掉，只留下 span 內的文字。

### 5. 後記格式

AO3 匯出的 afterword：

```html
<div id="afterword">
  ...
  <blockquote class="userstuff"><p>後記內容</p></blockquote>
</div>
```

整理成：

```md
> End Notes / 後記
> 後記內容
```

不保留 AO3 留言提示：

```html
Please drop by the Archive and comment...
```

## 檔案對應修改清單

### `docs/比對/浴室裡的相遇.md`

對應範本：`docs/範本/浴室裡的相遇.md`

目前狀態：

- 還是完整 AO3 HTML 匯出格式。
- 有 `preface`、`head/style`、`afterword`、AO3 留言連結。
- 正文每段包著 `<span>`，並且每段後面有空白 span。
- 正文仍是英文原文。

應修改為：

```md
---
title: 【佐久侑】浴室裡的相遇
author: Bryttdaffodil
summary: |
  > 多年來，侑始終沒有接住那些暗示、凝視，還有明目張膽的調情。於是佐久早決定親自出手，把事情，也把侑，掌握在自己手裡。
---

<div id="chapters" class="userstuff">
  <div class="userstuff">
    <p>翻譯後正文。</p>
  </div>
</div>

> End Notes / 後記
> 然後他們從此過著幸福快樂的日子。或差不多就是那樣。
```

處理重點：

- 刪除第 1 行到 `<body>` 的所有 HTML 外殼。
- 刪除 `<div id="preface">...</div>`。
- 刪除 `<h2 class="toc-heading">Bathroom (en)counter</h2>`。
- 將 `<span>英文正文</span>` 改成 `<p>繁中翻譯正文</p>`。
- 刪除空白 `<span>`。
- 將 `<div id="afterword">` 轉成 blockquote 後記。
- 刪除 AO3 comment 連結。

- 每章正文已經有 `<!--chapter content-->` 標記，但外層仍是 AO3 結構。

### Chapter 4

<!--chapter content-->

處理重點：

## 最後檢查清單

修改完每份檔案後，逐項確認：

- 檔案第一行是 `---`。
- frontmatter 有 `title`、`author`、`summary`。
- frontmatter 結束後有第二個 `---`。
- 單章作品只保留一組 `div id="chapters"`。
- 多章作品每章都有 `### Chapter N`。
- 多章作品每章都有一組 `<!--chapter content-->` 與 `<!--/chapter content-->`。
- 沒有 `<!DOCTYPE>`、`<html>`、`<head>`、`<body>`。
- 沒有 `<style>`。
- 沒有 `<div id="preface">`。
- 沒有 AO3 comment 提示文字。
- 沒有空白 `<span>`。
- 沒有只有 `?` 的段落。

## 建議工作流程

1. 從 `docs/比對` 複製原始檔到正式小說資料夾或新檔案。
2. 先刪除 HTML 外殼與 preface。
3. 補上 frontmatter。
4. 依單章或多章套用對應章節格式。
5. 清掉 span 與空白段落。
6. 處理後記。
7. 用搜尋檢查是否還有 `DOCTYPE`、`preface`、`afterword`、`toc-heading`、`span`。
8. 最後再打開 `docs/範本/範本01.md` 對照一次，確認章節標題、`<!--chapter content-->` 標記、`<div>` 包法與 frontmatter 排版一致。

## 小說清單 JSON 建立規則

建立 `list.json` 或補入資料集時，每篇小說使用同一種物件格式：

```json
{
  "id": "作品名",
  "title": "【CP】作品名",
  "author": "真實作者",
  "tags": ["英文CP標籤"],
  "file": "世界分類/英文CP標籤/真實檔案名稱.md",
  "description": "frontmatter summary 內容",
  "rating": null
}
```

欄位規則：

- `id`：使用去掉 `【CP】` 後的作品名。
- `title`：保留完整標題，例如 `【佐久侑】服從嚐起來像蜂蜜`。
- `author`：優先填寫 markdown frontmatter 或 AO3 byline 裡的真實作者；找不到資料時才填 `"未知"`。
- `tags`：使用對應 CP 的英文 `cpKey`，來源參照 `public/data/word.json`。
- `file`：路徑依不同作品世界分類調整，例如 `Haikyuu/SakuAtsu/Obedience_tastes_like.md`。
- `file` 的檔案名稱必須抓真實存在的 markdown 檔名，不要自行改成中文作品名。
- 若需要產生安全檔名，空白轉底線，並移除 Windows / URL 不合法字元。
- `description`：填入該 markdown frontmatter 裡的 `summary:` 內容，移除每行前面的 `>` 符號。
- `rating`：固定填 `null`。

CP 對應英文標籤請以 `public/data/word.json` 為準，不要只固定 DrStone。常用例：
